**Claude Code Daily** — 2026-04-12

**TL;DR:** Anthropic launched Claude Managed Agents in public beta, positioning itself as the managed harness host while third-party harnesses remain locked out — a contrast the community isn't missing. The AMD performance regression crisis has a root cause now (thinking-content redaction), and Zscaler published specific CVEs from the source leak that developers should action immediately.

---

**Big News**

- **Claude Managed Agents public beta** — Anthropic's fully managed agent hosting launched April 8 at $0.08/runtime hour (plus token costs). It provides secure sandboxing, long-running sessions, scoped permissions, and built-in tool execution so you don't build your own agent loop. Early customers include Notion, Rakuten, Asana, and Sentry. The companion `ant` CLI lets you version API resources as YAML and integrates natively with Claude Code. Notably ironic timing: Anthropic is now the managed harness provider while blocking third-party ones. [Anthropic blog](https://claude.com/blog/claude-managed-agents) · [The New Stack](https://thenewstack.io/with-claude-managed-agents-anthropic-wants-to-run-your-ai-agents-for-you/) · [Docs](https://platform.claude.com/docs/en/managed-agents/overview)

- **AMD regression crisis has a root cause** — Extensive independent analysis of GitHub issue #42796 (6,852 sessions, 234,760 tool calls, 17,871 thinking blocks) now points to `redact-thinking-2026-02-12` as the culprit. When Anthropic started hiding thinking-block content from session logs, reads-per-edit collapsed from 6.6x to 2.0x and blind edits jumped from 6.2% to 33.7%. AMD has already migrated to a competitor. No official Anthropic response yet. [GitHub issue](https://github.com/anthropics/claude-code/issues/42796) · [WinBuzzer](https://winbuzzer.com/2026/04/07/amd-ai-director-claude-code-performance-decline-7000-sessions-xcxwbn/) · [Lilting deep-dive](https://lilting.ch/en/articles/claude-code-quality-regression-thinking-redaction)

- **Zscaler publishes CVEs from the source leak** — ThreatLabz released a detailed security analysis identifying CVE-2025-59536 (RCE via malicious repo clone) and CVE-2026-21852 (API key exfiltration). These were pre-existing bugs now made trivially exploitable with the leaked source. If you work on AI tooling or CI/CD pipelines that invoke Claude Code, patching should be treated as urgent. [Zscaler ThreatLabz](https://www.zscaler.com/blogs/security-research/anthropic-claude-code-leak) · [CyberSecurityNews](https://cybersecuritynews.com/claude-code-leak-to-spread-vidar-and-ghostsocks-malware/)

**Power User Tips**

- **`/team-onboarding` generates a ramp-up guide from your usage** — New in v2.1.101 (April 10): run `/team-onboarding` and Claude Code analyzes your local session history to produce a tailored onboarding doc for new teammates. Great for async teams and open-source contributors. [ClaudeWorld](https://claude-world.com/articles/claude-code-21101-release/) · [Daily1Bite guide](https://daily1bite.com/en/blog/ai-tutorial/claude-code-april-2026-update)

- **Enterprise TLS proxies now work by default** — Claude Code now trusts the OS CA certificate store, so corporate TLS inspection proxies no longer break the CLI. Previously required manual cert configuration that blocked many enterprise teams. Just update and it works.

- **"Team OS" workflow with sub-agent roles** — A DoorDash PM's newsletter post on building a Team OS in Claude Code is getting shared widely: assign distinct `.claude/agents/` personas (architect, reviewer, QA lead) and route tasks explicitly. Concrete patterns for larger-scale agentic work. [Newsletter](https://www.news.aakashg.com/p/claude-code-team-os)
