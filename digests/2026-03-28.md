**Big News**

Max 5x users say their 5-hour windows drain in ~90 minutes; one Max 20x user watched usage jump from 21% → 100% on a single prompt. Anthropic has not publicly commented. Likely tied to Opus 4.6 token usage. If you're hitting limits unusually fast, you're not alone.

- **[Spring Break 2x promotion ends TONIGHT](https://support.claude.com/en/articles/14063676-claude-march-2026-usage-promotion)** — March 28 at 11:59 PM PT is the hard cutoff. Doubles usage on Free, Pro, Max, and Team plans outside peak hours (8 AM–2 PM ET weekdays). After tonight, quotas snap back to standard levels.

- **[Third-party client impersonators blocked](https://alphaguruai.substack.com/p/whats-going-on-with-claude-code)** — Anthropic has been shutting down apps that spoofed the official Claude Code client to run automated sessions under flat subscription pricing. High-volume users using these tools are now being pushed to the commercial API.

**Power User Tips**

- **v2.1.86 (Mar 27): `X-Claude-Code-Session-Id` header** — All Claude Code API requests now carry a session ID header, letting proxy operators and log aggregators group requests by session without body parsing. Useful if you're building tooling on top of Claude Code's API traffic.

- **MCP tool descriptions capped at 2KB** — OpenAPI-generated MCP servers were notoriously bloating context with huge schema dumps. The new cap keeps things lean. If your server was exceeding this, trim server instructions or description fields.

- **VSCode rate limit warning** — The extension now shows a banner with your current usage percentage and quota reset time — so you can see exactly how close you are before a session grinds to a halt.

- **`Ctrl+S` in `/stats` is 16x faster** — Screenshot capture in the stats panel has been dramatically sped up. Works in all builds now (was broken in some).
