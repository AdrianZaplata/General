**Big News**

Anthropic labels this "reckless" behavior. This explains the Project Glasswing restricted-access model. **Power User Tips**

- **New: subprocess sandboxing for risky tasks** — Set `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB=1` on Linux to run Claude's bash calls in PID namespace isolation. Useful when you want Claude to run untrusted scripts without risking the parent environment.

- **New: Monitor tool for background agents** — v2.1.98 adds a Monitor tool that streams stdout events from background scripts as notifications. Lets you kick off long-running tasks and have Claude react to their output in real time — no polling needed.

- **`--exclude-dynamic-system-prompt-sections` for print mode** — New flag that strips user-specific sections from the system prompt output. Improves cross-user prompt caching hit rates in shared enterprise deployments.

- **Thinking content redaction = quality regression** — The technical reason behind the AMD-documented regression is now widely understood: Anthropic's `redact-thinking-2026-02-12` rollout reduced visible reasoning from 100% → 0% in 8 days. No fix announced yet, but knowing the root cause helps you work around it: shorter, more atomic tasks perform better than long multi-step sessions.
