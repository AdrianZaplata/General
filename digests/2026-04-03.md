**Claude Code Daily** — 2026-04-03

Meanwhile, v2.1.91 shipped real improvements — MCP result sizes up to 500K and new plugin executable support.

**Big News**
- **[Critical vulnerability post-leak](https://www.securityweek.com/critical-vulnerability-in-claude-code-emerges-days-after-source-leak/)** — A security flaw in Claude Code surfaced days after the March 31 npm source map leak. Timing suggests the exposed source helped researchers spot it faster. Update to v2.1.91 immediately.
- **[Proactive mode confirmed in leaked source](https://www.engadget.com/ai/claude-code-leak-suggests-anthropic-is-working-on-a-proactive-mode-for-its-coding-tool-150107049.html)** — Engadget digs into the KAIROS/Proactive Mode codebase: an always-on mode where Claude monitors your environment and acts without being prompted. Not yet shipped; no ETA from Anthropic.
- **v2.1.91 released (April 2)** — MCP tool result persistence override added: annotate results with `_meta["anthropic/maxResultSizeChars"]` (up to 500K) to pass large payloads like full DB schemas without truncation. Also: plugins can now ship executables under `bin/` and invoke them as bare commands from Bash.
- **[Message Batches API max_tokens raised to 300K](https://platform.claude.com/docs/en/release-notes/overview)** — Applies to Opus 4.6 and Sonnet 4.6. Useful for long-form content, large structured data, and bulk code generation jobs in batch pipelines.

---

**Power User Tips**
- **Bypass MCP truncation for big schemas** — Use `_meta["anthropic/maxResultSizeChars"]` in your MCP server's tool result to tell Claude Code not to truncate. Up to 500K chars now allowed. Critical for database-heavy workflows where schema context kept getting dropped.
- **Deferred permissions in headless mode (v2.1.89)** — `PreToolUse` hooks can now return `"defer"` to pause a headless session mid-run. Resume it later with `-p --resume`. Clean solution for human-in-the-loop approval flows without killing the session.
- **Fix `--resume` prompt-cache misses** — v2.1.90 fixed a regression where `--resume` caused full cache misses on first request for users with deferred tools. If you do a lot of long-session resumptions, updating to 2.1.90+ is worth it.
- **Multi-line deep links now work** — `claude-cli://open?q=` deep links now accept encoded newlines (`%0A`). Previously only single-line prompts worked, limiting automation from external apps and scripts.
