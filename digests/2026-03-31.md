**Big News**
- **[Claude Mythos leaked](https://fortune.com/2026/03/26/anthropic-says-testing-mythos-powerful-new-ai-model-after-data-leak-reveals-its-existence-step-change-in-capabilities/)** — A CMS config error exposed Anthropic's draft blog post for an unreleased model described as their most powerful ever, sitting above Opus in a new tier called "Capybara." Anthropic confirmed it's in limited testing. The leak flagged "unprecedented cybersecurity risks" from the model's ability to find and exploit vulnerabilities.
- **[Claude loses >99% uptime in Q1 2026](https://news.ycombinator.com/item?id=47543189)** — A notable outage this week ended Anthropic's near-perfect uptime streak. Worth tracking if you depend on Claude Code in CI/CD.
- **March 30 release** — `CLAUDE_CODE_NO_FLICKER=1` env var opts into flicker-free alt-screen rendering with virtualized scrollback. A new `PermissionDenied` hook fires after auto mode classifier denials — useful for logging or alerting when Claude hits a wall.

**Power User Tips**
- **[How I'm Productive with Claude Code](https://news.ycombinator.com/item?id=47494890)** (HN, 1 day ago) — Top techniques: spawn multiple agents to implement the same requirement independently then cherry-pick the best parts; aim agents at docs to extract answers; use agents to review code and build reusable checklists.
- **OpusPlan workflow** — Switch to Opus 4.6 for planning, Sonnet 4.6 for execution. Keeps quality high on complex tasks while cutting token costs on the grunt work.
- **lessons.md pattern** — Every time Claude makes a mistake and you correct it, have it add a rule to `lessons.md`. Over time Claude self-corrects for your specific project's quirks.
- **[Claude Code Cheat Sheet discussion](https://news.ycombinator.com/item?id=47495527)** (HN, 4 days ago) — Note: prompting Claude to list "common and useful" commands may cause it to skip anything it classifies as dangerous. Be explicit if you need those included.
