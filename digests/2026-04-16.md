**Claude Code Daily** — 2026-04-16

**TL;DR:** Anthropic shipped a ground-up desktop app redesign built around running parallel Claude Code sessions side by side, and quietly launched Ultraplan — a cloud-hosted planning mode that offloads long thinking to a remote container while your terminal stays free. Meanwhile, the week-long performance-degradation controversy earned mainstream tech press coverage.

---

**Big News**

- **Desktop app rebuilt for parallel sessions** — The Claude desktop app got a full redesign (announced April 15) centered on running multiple Claude Code sessions simultaneously. Key additions: a sidebar showing all active and recent sessions with filter/group by project, drag-and-drop layout, an integrated terminal, in-app file editor, rebuilt diff viewer for large changesets, and an expanded preview pane (HTML, PDF, local servers). Three view modes: Verbose, Normal, Summary. Mac now supports SSH for remote machines. Available on all paid plans + API. [Anthropic Blog](https://claude.com/blog/claude-code-desktop-redesign) · [MacRumors](https://www.macrumors.com/2026/04/15/anthropic-rebuilds-claude-code-desktop-app/) · [Thurrott](https://www.thurrott.com/a-i/anthropic/334911/anthropic-redesigns-claude-app-on-desktop-for-parallel-agents)

- **Ultraplan: offload planning to the cloud (early preview)** — `/ultraplan` hands your planning task to a remote Opus 4.6 container for up to 30 minutes while your terminal stays free. You review the plan in a browser editor with inline comments, emoji reactions, and an outline sidebar; then execute it locally or in the cloud. Requires v2.1.91+, a Claude Code web account, and a GitHub repo. Not available on Bedrock/Vertex/Foundry. [Claude Docs](https://code.claude.com/docs/en/ultraplan) · [The Decoder](https://the-decoder.com/claude-codes-new-ultraplan-feature-moves-task-planning-to-the-cloud/)

- **Anthropic pledges no ads — ever** — On April 14, Anthropic published a statement committing to keeping Claude ad-free, arguing that advertising incentives are fundamentally incompatible with a genuinely helpful AI assistant. [Anthropic News](https://www.anthropic.com/news)

**Power User Tips**

- **Boris Cherny's parallel-worktree pattern** — Claude Code's creator runs 10–15 sessions simultaneously by giving each its own git worktree so changes never collide. If you're using the new multi-session desktop UI, pair it with `git worktree add` per session to avoid merge chaos.

- **Two-Claude plan review** — Before any coding session: open Plan Mode, have Claude 1 draft a plan, then open a second session and ask Claude 2 to review it "as a staff engineer." Only execute after both agree. Consistently cited as the highest-leverage single change to output quality. [Claude Help Center](https://support.claude.com/en/articles/14554000-claude-code-power-user-tips)

- **Custom subagents in `.claude/agents/`** — Drop `.md` files into `.claude/agents/` with a name, tool allowlist, permission mode, and model to create reusable PR-workflow automations. Each agent can target a different model (e.g. Haiku 4.5 for cheap triage, Opus 4.6 for deep refactors). [awesome-claude-code](https://github.com/hesreallyhim/awesome-claude-code)
