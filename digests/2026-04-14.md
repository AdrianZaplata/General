**Claude Code Daily** — 2026-04-14

**TL;DR:** Anthropic had a huge week: Claude Managed Agents launched in public beta (April 8), Claude Mythos Preview debuted as a security-focused powerhouse model (April 7), and Claude for Word went into beta (April 11). Meanwhile Anthropic officially confirmed the performance regression that AMD's Stella Laurenzo exposed — two deliberate product changes were to blame.

---

**Big News**

- **Claude Managed Agents public beta** — Launched April 8, this is a fully managed agent harness: secure sandboxed containers, session persistence, tool orchestration, and multi-agent coordination via API. Developers define tasks and guardrails; Anthropic handles the infra. Billed at $0.08/runtime hour + standard model costs. Works via Console, Claude Code, and a new CLI. [SiliconANGLE](https://siliconangle.com/2026/04/08/anthropic-launches-claude-managed-agents-speed-ai-agent-development/) · [Winbuzzer](https://winbuzzer.com/2026/04/10/anthropic-launches-claude-managed-agents-enterprise-ai-xcxwbn/)

- **Claude Mythos Preview: Anthropic's cybersecurity specialist model** — Announced April 7, Mythos Preview autonomously found and exploited a 17-year-old RCE vulnerability in FreeBSD (CVE-2026-4747), giving an attacker full server control. Anthropic immediately launched Project Glasswing to use Mythos to secure critical software. Strongest model yet at security tasks. [red.anthropic.com](https://red.anthropic.com/2026/mythos-preview/)

- **Claude for Word Beta is live** — Anthropic's Claude add-in for Microsoft Word launched in beta on April 11. Write, edit, and summarize directly inside Word without switching tools. [WPS Blog](https://www.wps.com/blog/claude-for-word-beta/)

- **Anthropic confirms regression: adaptive thinking + lowered effort caused it** — After AMD's Stella Laurenzo published 6,852-session data showing ~67% regression in complex tasks, Anthropic officially confirmed the root cause: adaptive thinking and a deliberately lowered default effort setting. Not a bug — intentional product changes that reduced reasoning depth for power users. Community reaction: not great.

- **Computer use lands in Claude Code CLI** — Claude can now open native desktop apps, click through UI, test its own code changes, and fix what breaks — all from your terminal. No browser required.

**Power User Tips**

- **`/team-onboarding` generates a teammate ramp-up guide** — New command that analyzes your local Claude Code usage patterns and produces a tailored onboarding doc for new engineers joining your project. Run it before your next hire's first day.

- **Subagents via `.claude/agents/`** — Drop `.md` files into `.claude/agents/` to define specialized agents Claude can delegate to for isolated tasks (each gets its own context + allowed tools). Think of them as automations for common PR workflows, code review passes, or test runs. [Claude Code Docs](https://code.claude.com/docs/en/best-practices)

- **Context architecture is the real leverage point in 2026** — The model is rarely the bottleneck; your context is. Power users are spending more time on context engineering (what goes in, what stays out, when to compact) than on prompts. [Towards AI](https://medium.com/@richardhightower/claude-code-2026-the-daily-operating-system-top-developers-actually-use-d393a2a5186d)

- **Google Vertex AI now has an interactive setup wizard** — Select "3rd-party platform" at login to get a guided GCP auth → project → region → credential check → model pinning flow. No more manual config for enterprise teams.
